# リポジトリガイドライン

## AIエージェントとの協業方針

- このリポジトリは Rust / TanStack Start 学習を主目的とし、実装の主体はユーザー（実装者）とする。
- AIエージェントは原則としてレビュー担当とし、コードの問題点・改善点・リスクを指摘する。
- AIエージェントは、ユーザーから明示的に依頼された場合に限り、部分的な実装や修正を行う。
- 実装依頼がない場合は、設計相談、実装方針の提案、レビューコメント、学習のための解説を優先する。

## ドキュメントの分類

### 1. 永続的ドキュメント（`docs/`）

作業にあたっては、以下のドキュメントを参照することを推奨する。これらはプロジェクト全体のルールや方針を示すもので、作業引き継ぎドキュメントとは別に管理する。

- `01_product_requirements.md`
    - プロダクト要求仕様書
    - 主要な機能一覧
- `02_architecture.md`
    - アーキテクチャ設計書
    - レイヤ構成と責務
- `03_development_guidelines.md`
    - コーディングスタイル
    - テスト指針
    - コミット・PR指針
- `04_software_design.md`
    - backend/frontend の設計上のポイント

### 2. 作業引き継ぎドキュメント（`.steering/`）

作業単位のドキュメントを `.steering/` ディレクトリに保存する。これらは作業の経過や内容を記録するもので、プロジェクト全体のルールや方針を示すものではない。

- 最新のドキュメントは `.steering/latest.md` として残す。
- 過去のドキュメントは `.steering/{年}{月}{日}_{時}{分}.md` として残し、変更しない。
- 読み込む時は常に最新のドキュメントを読み込み、書き込む時はその時点での最新のドキュメントを過去のドキュメントとして保存してから最新のドキュメントを更新する。
